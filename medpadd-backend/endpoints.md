# MedPadd Backend API Endpoints

## Core Entities

### System Roles

**Base URL:** `/api/system-roles`

- `GET /api/system-roles` - Get all system roles
- `GET /api/system-roles/{id}` - Get system role by ID
- `POST /api/system-roles` - Create new system role
- `PUT /api/system-roles/{id}` - Update system role
- `DELETE /api/system-roles/{id}` - Delete system role

## Notes

- All IDs are UUIDs
- Request/Response bodies are in JSON format
- Successful responses return 200 OK (or 204 No Content for DELETE)
- Not found resources return 404 Not Found
- Created resources return the created entity in response body
